# MineDeck

**Лёгкая веб-панель для управления одним Minecraft-сервером.**
Консоль, файловый менеджер, бэкапы, таймеры, настройка ОЗУ, портов и фаервола — в одном красивом интерфейсе.

Разворачивается **одной командой**. Без нод, без агентов, без базы данных — только тот сервер, на котором стоит панель.

> Вдохновлено удобством [Pterodactyl](https://pterodactyl.io/), но переосмыслено под **один сервер**: тот же уровень контроля (консоль, файловый менеджер, лимиты ресурсов, бэкапы), но без сложной инфраструктуры — панели, демона Wings, нод и БД.

## ✨ Возможности

- 🖥️ **Живая консоль** — вывод сервера в реальном времени (WebSocket), ввод команд с **историей** (↑/↓) и **автодополнением по Tab** (стандартные команды 1.20.1, аргументы и ники онлайн-игроков).
- ▶️ **Управление процессом** — запуск / стоп / рестарт / принудительное завершение, авто-перезапуск при падении, автозапуск.
- 📊 **Мониторинг** — CPU и ОЗУ процесса сервера, ОЗУ и диск системы, аптайм.
- 🗂️ **Файловый менеджер (в стиле Pterodactyl)** — навигация по папкам (клик/двойной клик), редактор текстовых файлов, **загрузка перетаскиванием (файлы и целые папки)**, а действия (скачать, переименовать, удалить) собраны в контекстное меню «три точки» справа. Скачивание папок — как `.tar.gz`.
- 🧮 **Ограничение CPU** — выделите серверу нужное число **vCPU** (ядер); привязка к ядрам через `taskset`, чтобы игровой сервер не «съедал» весь процессор.
- 🗄️ **Базы данных (MySQL / MariaDB)** — создание базы с отдельным пользователем и паролем в один клик (готовые данные для конфига плагина), список с размерами, удаление и встроенная **SQL-консоль** прямо в панели.
- ⬆️ **Обновление из репозитория** — обновление панели до последней версии с GitHub в один клик прямо из интерфейса (с сохранением входа и настроек).
- 🗄️ **Бэкапы** — резервные копии всей директории сервера в один клик, восстановление, авто-очистка старых, скачивание.
- ⏱️ **Таймеры** — плановые задачи: ежедневный рестарт, интервальные бэкапы, команды по расписанию, с предупреждением игроков.
- 🧠 **Настройка ОЗУ и запуска** — `-Xms`/`-Xmx`, флаги Aikar, путь к Java, произвольная команда запуска, предпросмотр.
- ⚙️ **server.properties** — удобный редактор основных и всех параметров (порт, MOTD, сложность, вайтлист и т.д.).
- 🔥 **Фаервол** — управление `ufw`: включение, правила allow/deny по портам.
- 🌐 **Порт панели** и **смена пароля** прямо из интерфейса.
- 🔐 **Авторизация** — вход по логину и паролю (bcrypt), первый запуск создаёт администратора.

## 🚀 Установка (Ubuntu 22.04)

Одна команда — устанавливает зависимости (Node.js, Java, ufw), скачивает панель с GitHub и запускает её как systemd-сервис:

```bash
curl -fsSL https://raw.githubusercontent.com/denny4-user/minedeck/main/install.sh | sudo bash
```

После установки откройте в браузере:

```
http://<IP-вашего-сервера>:8080
```

При первом входе панель предложит создать логин и пароль администратора.

> 💡 **Обновление** — прямо из панели: **Настройки → Панель → Проверить обновления → Обновить сейчас**. Панель подтянет свежую версию из GitHub и перезапустится (вход и настройки сохранятся). Либо запустите ту же команду установки ещё раз.

## 🎮 Быстрый старт с Minecraft-сервером

1. Откройте панель и войдите.
2. Перейдите в **Файлы** и загрузите ваш `server.jar` в директорию сервера (по умолчанию `/opt/minecraft`).
3. В **Настройки → Сервер и ОЗУ** укажите имя JAR-файла и выделяемую память, примите **EULA**.
4. Нажмите **▶ Запуск** в верхней панели — следите за загрузкой в консоли.
5. В **Настройки → Фаервол** откройте игровой порт (по умолчанию `25565/tcp`).

## 🛠️ Управление сервисом

```bash
systemctl status minedeck      # статус
systemctl restart minedeck     # перезапуск (после смены порта панели)
systemctl stop minedeck        # остановить панель
journalctl -u minedeck -f      # логи панели
```

Расположение по умолчанию:

| Что | Путь |
|-----|------|
| Код панели | `/opt/minedeck` |
| Конфиг и данные | `/opt/minedeck/data/config.json` |
| Бэкапы | `/opt/minedeck/data/backups` |
| Файлы Minecraft-сервера | `/opt/minecraft` |

## 🔒 Безопасность

- Панель работает от `root` (нужно для управления `ufw` и файлами сервера). Разворачивайте на доверенной машине.
- Доступ защищён логином/паролем. Пароль хранится в виде bcrypt-хэша.
- Рекомендуется ограничить доступ к порту панели фаерволом или проксировать через HTTPS (nginx + Let's Encrypt).
- Перед включением `ufw` **обязательно** разрешите порт SSH (22) и порт панели, иначе потеряете доступ.

## 🧩 Технологии

Node.js + Express + WebSocket (`ws`) на бэкенде. Фронтенд — HTML/JS без сборки на сервере;
интерфейс на **Tailwind CSS + daisyUI** (тёмная тема) с иконками **Lucide**, собранный CSS
(`public/css/app.css`) закоммичен, поэтому на сервере сборка не нужна и панель работает офлайн.
Рантайм-зависимости: `express`, `express-session`, `ws`, `bcryptjs`, `multer`, `mysql2` (для БД).

## 🧑‍💻 Запуск для разработки

```bash
git clone https://github.com/denny4-user/minedeck.git
cd minedeck
npm install
npm start           # http://localhost:8080
```

Если правите интерфейс — пересоберите стили (Tailwind/daisyUI в devDependencies):

```bash
npm run build:css   # собрать public/css/app.css
npm run watch:css   # пересобирать на лету при разработке
```
